# Домашнє завдання: Графи та алгоритми пошуку шляхів

## Завдання 1: Створення та аналіз графа
- Було створено граф з **26 вузлів** та **34 ребер**.
- Граф є **зв’язним**.
- Приклад аналізу вузла:
  - **Kyiv**: степінь 5, сусіди — Zhytomyr, Chernihiv, Cherkasy, Poltava, Vinnytsia.
- Аналіз ребер показав, що прямого з'єднання між Києвом та Львовом немає.
- Найкоротший шлях між Kyiv та Lviv (по кількості вузлів): `['Kyiv', 'Zhytomyr', 'Rivne', 'Lviv']`.

## Завдання 2: Пошук шляхів алгоритмами DFS і BFS
- Алгоритми DFS і BFS реалізовані для побудованого графа.
- При порівнянні шляхів **Kyiv → Lviv** обидва алгоритми знайшли один і той самий шлях: `Kyiv -> Zhytomyr -> Rivne -> Lviv`.
- При порівнянні шляхів **Odesa → Luhansk** алгоритми дали різні результати:
  - DFS: `Odesa -> Mykolaiv -> Kropyvnytskyi -> Cherkasy -> Kyiv -> Poltava -> Dnipro -> Zaporizhzhia -> Donetsk -> Kharkiv -> Luhansk` (довжина 11 вузлів)
  - BFS: `Odesa -> Mykolaiv -> Kropyvnytskyi -> Dnipro -> Kharkiv -> Luhansk` (довжина 6 вузлів)
- Різниця пояснюється особливостями алгоритмів: DFS йде «углиб», BFS знаходить мінімальну кількість кроків.

## Завдання 3: Алгоритм Дейкстри для зваженого графа
- Графу має ваги ребер (відстань у км).
- Найкоротші шляхи за сумарною відстанню:
  - **Kyiv → Lviv**: `Kyiv -> Zhytomyr -> Rivne -> Lviv`, **540 км**
  - **Odesa → Luhansk**: `Odesa -> Mykolaiv -> Kherson -> Zaporizhzhia -> Dnipro -> Kharkiv -> Luhansk`, **1090 км**
- Результати збігаються з результатами NetworkX для перевірки.
- Алгоритм Дейкстри дозволяє знайти найкоротший шлях за відстанню, що відрізняється від мінімальної кількості кроків, яку дає BFS.
